HTML

  • Hypertext Markup Language (HTML) is used to create the stucture of a webpage.
  • When we create the structure for our webpage, we used a series of HTML elements to define the content framework.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • An HTML Attribute defines additionla information about an element.
  • HTML Attributes are always defined in the begining elemt, never the closing element.
  • Elements that are nested inside other elements are called child element.
  • The element that holds all the children is knownn as the parent element.
  • In Semantic HTML elements , the name of the element describes its content an the role it plays on a webpage.
  • Semantic HTML elements make your HTML easier to read by other developers and the browser.
  • An element refers to the entire element from opening tag to closing tag
  • Elements give structure to the content on the webpage.
  • Tag refers only to what appears inside the angle brackets.

Git

  • Git & GitHub is a version control software and cloud hosting service that let us keep track of changes we make to our code.
  • To access Git use the command line
  • Git commands always start with "git" and are used to save and track version of your code.
  • A branch is a copy or version of the code that can be worked on without affecting the default branch.
  • Our default branch is called "main". We should'nt do any coding directly in the "main" branch.
  • To make changes to the main branch. we should create a feature branch.
  • A feature branch is a copy of the main branch.
  • In the feature branch, we make commits of all changes we make to the code while working.
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
